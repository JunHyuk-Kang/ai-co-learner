# 역량 평가 객관식 질문 20가지

AI Co-Learner 초기 진단 시스템에서 사용할 객관식 역량 평가 질문입니다.
사용자의 6가지 핵심 역량을 측정하기 위해 설계되었습니다.

---

## 역량 매핑

각 질문은 하나 이상의 역량을 측정합니다:

- **Q**: 질문력 (Question Quality)
- **T**: 사고력 (Thinking Depth)
- **C**: 창의력 (Creativity)
- **Co**: 소통력 (Communication Clarity)
- **E**: 실행력 (Execution Oriented)
- **Cl**: 협업력 (Collaboration Signal)

---

## 질문 목록

### Q1. 문제 해결 접근법 [T, E]
**질문**: "코딩 문제를 처음 봤을 때, 가장 먼저 무엇을 하나요?"

**선택지**:
1. (1점) 바로 코드를 작성하기 시작한다
2. (2점) 비슷한 문제를 검색해서 참고한다
3. (3점) 문제를 작은 단위로 쪼개고 단계별로 계획을 세운다
4. (2점) 다른 사람에게 물어본다

**측정 역량**: 사고력(T), 실행력(E)

---

### Q2. 에러 대처 방식 [E, Q]
**질문**: "코드 실행 중 에러가 발생했을 때, 어떻게 대처하나요?"

**선택지**:
1. (1점) 에러를 무시하고 다른 방법을 시도한다
2. (2점) 에러 메시지를 복사해서 검색한다
3. (3점) 에러 메시지를 분석하고, 디버거나 로그로 원인을 추적한다
4. (2점) 처음부터 다시 작성한다

**측정 역량**: 실행력(E), 질문력(Q)

---

### Q3. 새로운 기술 학습 [E, T]
**질문**: "새로운 프로그래밍 언어나 프레임워크를 배울 때, 어떤 방법을 선호하나요?"

**선택지**:
1. (2점) 온라인 강의를 처음부터 끝까지 시청한다
2. (3점) 공식 문서를 읽고 작은 프로젝트를 만들어본다
3. (1점) 필요할 때마다 검색해서 배운다
4. (2점) 튜토리얼을 따라하며 익힌다

**측정 역량**: 실행력(E), 사고력(T)

---

### Q4. 코드 품질 인식 [T, Co]
**질문**: "좋은 코드란 무엇이라고 생각하나요?"

**선택지**:
1. (1점) 짧고 간단한 코드
2. (2점) 버그가 없고 잘 작동하는 코드
3. (3점) 읽기 쉽고 유지보수가 용이하며, 확장 가능한 코드
4. (2점) 빠르게 실행되는 코드

**측정 역량**: 사고력(T), 소통력(Co)

---

### Q5. 정보 검증 태도 [T, Q]
**질문**: "Stack Overflow나 블로그에서 찾은 코드를 어떻게 사용하나요?"

**선택지**:
1. (1점) 그대로 복사해서 사용한다
2. (2점) 작동하는지 확인하고 사용한다
3. (3점) 코드를 이해하고, 내 상황에 맞게 수정해서 사용한다
4. (1점) 최신 답변이면 믿고 사용한다

**측정 역량**: 사고력(T), 질문력(Q)

---

### Q6. 창의적 문제해결 [C, T]
**질문**: "같은 기능을 구현하는 여러 방법이 있을 때, 어떻게 선택하나요?"

**선택지**:
1. (1점) 가장 쉬운 방법을 선택한다
2. (2점) 가장 많이 사용되는 방법을 선택한다
3. (3점) 각 방법의 장단점을 비교하고, 프로젝트 요구사항에 맞는 방법을 선택한다
4. (2점) 처음 떠오른 방법을 사용한다

**측정 역량**: 창의력(C), 사고력(T)

---

### Q7. 설명 능력 [Co, T]
**질문**: "기술적인 개념을 비전공자에게 설명해야 한다면?"

**선택지**:
1. (1점) 전문 용어를 그대로 사용해서 정확하게 설명한다
2. (2점) 최대한 쉬운 말로 바꿔서 설명한다
3. (3점) 일상적인 비유와 예시를 들어 단계적으로 설명한다
4. (1점) 설명하기 어려우니 문서를 공유한다

**측정 역량**: 소통력(Co), 사고력(T)

3. (3점) 서로의 코드를 검토하고 토론해서 더 나은 방법을 찾는다
4. (1점) 충돌을 피하기 위해 내 부분만 작업한다

**측정 역량**: 협업력(Cl), 소통력(Co)

**선택지**:
1. (1점) 기분이 나쁘고 방어적으로 반응한다
2. (2점) 요청된 부분만 수정한다
3. (3점) 피드백을 통해 배우고, 왜 그렇게 수정하는지 이해하려고 노력한다
4. (1점) 리뷰어가 틀렸다고 생각한다

**측정 역량**: 협업력(Cl), 실행력(E)

### Q10. 목표 설정 방식 [E, T]

## 점수 계산 방식

### 선택지별 점수
- **1점**: 소극적이거나 비효율적인 접근
- **2점**: 보통 수준의 접근
- **3점**: 적극적이고 효과적인 접근

### 역량별 점수 계산

각 역량별로 관련 질문들의 평균 점수를 계산하여 10점 만점으로 환산합니다.

```
역량 점수 = (관련 질문들의 합계 / 관련 질문 개수 / 3) * 10
```

## 결과 해석 가이드

### 각 역량별 점수 범위 해석 (10점 만점)

**8.0 - 10.0점**: 우수 - 해당 역량이 뛰어남
**6.0 - 7.9점**: 양호 - 평균 이상의 역량 보유
**4.0 - 5.9점**: 보통 - 개선 여지가 있음
**0.0 - 3.9점**: 부족 - 집중적인 학습 필요

## 활용 방법

### 1. 질문 랜덤화 (선택사항)
- 질문 순서를 랜덤화하여 매번 다른 경험 제공
- 단, 역량 분포가 균등하게 유지되도록 주의

### 2. 적응형 평가 (향후 개선)
- 초기 답변에 따라 난이도 조정
- 더 정확한 역량 측정 가능

### 3. 봇 추천 연계
- 역량 평가 결과를 기반으로 적합한 학습 봇 추천
- 약한 역량을 강화할 수 있는 학습 경로 제시
