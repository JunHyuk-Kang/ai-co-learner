# [아카이브] 역량 평가 객관식 질문 v1 - 자기평가형

> **폐기 사유**: SJT(상황판단검사) 방식으로 전면 교체 (2026-01-27)
> **교체 문서**: [../competency-assessment-questions.md](../competency-assessment-questions.md)

## 개요

자기평가형 20문항 객관식. 각 문항은 1개 역량에 매핑되며, 4개 선택지가 1~4점으로 고정 배점.

### 문제점 (교체 사유)
- **사회적 바람직성 편향**: 4번 선택지가 항상 "이상적 답변"이라 패턴 파악이 쉬움
- **변별력 부족**: 누구나 높은 점수를 받을 수 있어 실질적 역량 차이를 반영하지 못함
- **성향이 아닌 지식 측정**: "어떤 게 좋은 방법인지 아는가?"를 묻는 형태로, 실제 행동과 괴리

---

## 질문 목록 (기존 코드 원본)

### 질문력 (questionQuality) - 3문항

**Q1.** "문제를 해결할 때 나의 접근 방식은?"
1. (1점) 일단 시작하고 부딪히면서 배운다
2. (2점) 기본적인 질문 몇 가지를 생각해본다
3. (3점) 문제를 분석하고 핵심 질문들을 정리한다
4. (4점) 다각도로 질문을 만들어 문제의 본질을 파악한다

**Q2.** "새로운 주제를 학습할 때 나는?"
1. (1점) 주어진 내용을 그대로 받아들인다
2. (2점) 이해가 안 되는 부분만 질문한다
3. (3점) '왜 그럴까?'라는 질문을 자주 던진다
4. (4점) 연관된 질문들을 계속 만들어가며 깊이 탐구한다

**Q3.** "다른 사람의 의견을 들을 때 나는?"
1. (1점) 경청하고 받아들인다
2. (2점) 궁금한 점이 있으면 질문한다
3. (3점) 의견의 근거가 무엇인지 파악하려 한다
4. (4점) 질문을 통해 숨어있는 가정이나 전제를 발견한다

---

### 사고력 (thinkingDepth) - 4문항

**Q4.** "복잡한 문제를 마주했을 때 나는?"
1. (1점) 빠르게 해결책을 찾으려 한다
2. (2점) 여러 가지 방법을 시도해본다
3. (3점) 문제를 작은 부분으로 나누어 하나씩 분석한다
4. (4점) 문제가 왜 발생했는지 근본 원인부터 깊이 파고든다

**Q5.** "새로운 정보를 학습할 때 나의 방식은?"
1. (1점) 당장 필요한 핵심만 빠르게 파악한다
2. (2점) 중요한 내용을 정리하고 요약한다
3. (3점) 이미 알고 있는 지식과 연결지어 이해한다
4. (4점) 여러 관점에서 깊이 분석하고 실제로 어떻게 활용할지 고민한다

**Q6.** "중요한 결정을 내릴 때 나는?"
1. (1점) 직감을 믿고 빠르게 결정한다
2. (2점) 좋은 점과 나쁜 점을 간단히 비교한다
3. (3점) 여러 요소를 종합적으로 따져본다
4. (4점) 나중에 미칠 영향까지 생각하며 신중히 판단한다

**Q19.** "실패를 경험했을 때 나는?"
1. (1점) 아쉽지만 빨리 잊고 넘어간다
2. (2점) 무엇이 잘못되었는지 간단히 생각해본다
3. (3점) 실패 원인을 분석하고 교훈을 얻는다
4. (4점) 깊이 성찰해 근본 원인을 찾고 구체적인 개선 방안을 만든다

**Q20.** "처음 해보는 어려운 과제를 받았을 때 나는?"
1. (1점) 부담스럽고 걱정된다
2. (2점) 일단 해보면서 배운다
3. (3점) 필요한 것을 학습하고 준비해서 도전한다
4. (4점) 성장할 기회로 받아들이고 전략적으로 접근한다

---

### 창의력 (creativity) - 3문항

**Q7.** "문제 해결할 때 나의 아이디어 스타일은?"
1. (1점) 이미 검증된 안전한 방법을 따른다
2. (2점) 기존 방법을 조금 변형해본다
3. (3점) 여러 방법을 새롭게 조합해본다
4. (4점) 아무도 시도하지 않은 완전히 새로운 방법을 만든다

**Q8.** "일상에서 새로운 것을 접했을 때 나는?"
1. (1점) 익숙한 방식을 선호한다
2. (2점) 가끔 새로운 것을 시도한다
3. (3점) 자주 새로운 방식을 실험해본다
4. (4점) 항상 '이걸 다르게 하면 어떨까?' 상상한다

**Q9.** "아이디어를 떠올릴 때 나는?"
1. (1점) 대부분 사람들이 하는 일반적인 생각을 한다
2. (2점) 다른 사람의 좋은 아이디어를 참고한다
3. (3점) 서로 다른 분야의 아이디어를 섞어본다
4. (4점) 전혀 관련 없어 보이는 것들을 창의적으로 연결한다

---

### 소통력 (communicationClarity) - 3문항

**Q10.** "복잡한 내용을 다른 사람에게 설명할 때 나는?"
1. (1점) 내가 아는 대로 그대로 설명한다
2. (2점) 핵심 내용만 간단히 전달한다
3. (3점) 듣는 사람 수준에 맞춰 쉽게 풀어 설명한다
4. (4점) 일상 속 비유와 예시를 들어 누구나 이해하도록 전달한다

**Q11.** "내 의견을 전달할 때 나는?"
1. (1점) 내 생각을 말한다
2. (2점) 간결하게 핵심만 전달한다
3. (3점) 왜 그렇게 생각하는지 근거와 함께 설명한다
4. (4점) 상대방이 공감할 수 있게 논리적으로 구조화해서 전달한다

**Q12.** "글이나 발표 자료를 만들 때 나는?"
1. (1점) 생각나는 대로 내용을 나열한다
2. (2점) 중요한 부분을 강조한다
3. (3점) 논리적 흐름을 고려해서 구성한다
4. (4점) 메시지가 명확히 전달되도록 체계적으로 설계한다

---

### 실행력 (executionOriented) - 3문항

**Q13.** "계획을 세운 후 나는?"
1. (1점) 계획만 세우고 실행은 미루는 편이다
2. (2점) 시작은 하지만 끝까지 하지 못하는 경우가 많다
3. (3점) 대부분 계획한 것을 실행에 옮긴다
4. (4점) 즉시 행동하고 반드시 끝까지 완수한다

**Q14.** "목표를 달성하는 과정에서 나는?"
1. (1점) 흐름에 맡기고 자연스럽게 진행한다
2. (2점) 기본적인 단계를 밟아간다
3. (3점) 구체적인 실행 계획을 세우고 따라간다
4. (4점) 중간 목표를 정하고 계속 점검하며 적극적으로 추진한다

**Q15.** "새로운 프로젝트를 시작할 때 나는?"
1. (1점) 일단 머릿속으로만 생각해본다
2. (2점) 모든 준비가 완벽해지면 시작한다
3. (3점) 작은 것부터 빠르게 시작한다
4. (4점) 핵심 기능만 빠르게 만들어 테스트하고 계속 개선한다

---

### 협업력 (collaborationSignal) - 3문항

**Q16.** "팀 프로젝트에서 나는?"
1. (1점) 내가 맡은 역할만 수행한다
2. (2점) 필요할 때 다른 사람을 돕는다
3. (3점) 적극적으로 협력하고 의견을 조율한다
4. (4점) 팀 전체가 잘되도록 능동적으로 도우며 기여한다

**Q17.** "팀원과 의견이 충돌했을 때 나는?"
1. (1점) 내 의견이 옳다고 주장한다
2. (2점) 서로 양보할 점을 찾으려 한다
3. (3점) 서로의 입장을 이해하고 조율한다
4. (4점) 대화를 통해 더 나은 제3의 해결책을 함께 만든다

**Q18.** "팀원이 어려움을 겪는 것을 발견했을 때 나는?"
1. (1점) 각자 알아서 해결할 문제라고 생각한다
2. (2점) 도움을 요청하면 도와준다
3. (3점) 먼저 다가가 도움을 제안한다
4. (4점) 선제적으로 지원하고 함께 성장할 방법을 찾는다

## 채점 방식 (구버전)

```javascript
// 각 역량별 평균 → 1~10점 변환
const avgScore = scores[competency] / competencyCounts[competency];
scores[competency] = Math.round(((avgScore - 1) / 3) * 9 + 1);
```
