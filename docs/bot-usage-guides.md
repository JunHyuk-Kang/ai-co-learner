## 1. 창의력 에이전트

### 기본 정보
- **테마 색상**: 주황색 (Orange)
- **주요 역량**: 창의력 (Creativity)
- **보조 역량**: 질문력 (Question Quality)
- **ID**: t1764243446949

### 핵심 특징
이 봇은 학생의 아이디어를 **평가하지 않고** 다양한 방향으로 확장·변형하도록 돕습니다.
- **Yes, And 방식**: 학생의 모든 아이디어를 수용하고 확장
- **평가 금지**: "좋다/나쁘다" 판단하지 않음
- **학생 주도**: 확장 vs 전환 시점을 학생이 선택

### 어떤 학생에게 추천?
- 아이디어는 있지만 확장하는 데 어려움을 겪는 학생
- 정답을 찾으려는 고정관념이 강한 학생
- 창의적 사고의 폭을 넓히고 싶은 학생

### 활용 시나리오

#### 시나리오 1: 프로젝트 아이디어 확장
**상황**: 학생이 "급식 남기지 않기 캠페인"을 기획 중

```
학생: "남기는 사람은 벌칙을 주면 될 것 같아요."
봇: "그래, 남기는 행동을 줄이려는 생각이네. 그럼 벌칙 말고 전혀 다른 방향으로 바꾼다면?"

학생: "남기지 않으면 보상을 주는 거요."
봇: "좋아, 보상으로 방향이 바뀌었네. 그럼 이 보상을 사람이 아니라 공간에 주면?"

학생: "급식실이 깨끗해지면 장식이 바뀌는 거요."
봇: "괜찮다. 그럼 한 번만 더 확장해볼게. 이 장식이 매번 똑같지 않게 하려면?"
```

#### 시나리오 2: 주제 이탈 시 복귀
**상황**: 학생이 주제에서 벗어난 아이디어 제시

```
학생: "그냥 급식을 안 먹으면 되잖아요."
봇: "이 생각이 우리 문제랑 어떻게 연결될 수 있을까? 연결이 안 되면 하나만 바꿔볼까?"

학생: "아, 급식 메뉴를 학생들이 선택할 수 있게 하는 건 어때요?"
봇: "그래, 선택권으로 방향이 바뀌었네. 그럼 이 선택을 음식이 아니라 다른 요소에 적용한다면?"
```

### 교사 가이드

#### 사전 준비
1. **명확한 주제 설정**: 학생에게 탐색할 주제를 명확히 제시
2. **안전 범위 안내**: 다루지 말아야 할 주제 사전 공유 (폭력, 혐오 등)
3. **시간 제한**: 한 세션 15-20분 권장

#### 사후 활동
1. **아이디어 정리**: 학생이 확장한 아이디어들을 시각화 (마인드맵 등)
2. **평가는 교사가**: 봇은 평가하지 않으므로, 교사가 최종 피드백 제공
3. **실행 가능성 논의**: 확장된 아이디어 중 실현 가능한 것 선택

#### 주의사항
- ⚠️ 봇은 아이디어의 질을 평가하지 않습니다
- ⚠️ 비현실적인 아이디어도 수용합니다 (확장 훈련이 목적)
- ⚠️ 최종 선택과 평가는 반드시 교사 또는 학생이 해야 합니다

## 2. 사고력 에이전트

### 기본 정보
- **테마 색상**: 보라색 (Purple)
- **주요 역량**: 사고력 (Thinking Depth), 질문력 (Question Quality)
- **보조 역량**: 소통력 (Communication Clarity)
- **ID**: t1766115131611

### 핵심 특징
답을 주지 않고 계속 **"왜?"를 물으며** 비판적 사고와 논리 구축을 훈련합니다.
- **소크라테스 대화법**: 질문으로 스스로 답을 찾게 함
- **논리 오류 지적**: 성급한 일반화, 순환논증 등 지적
- **근거 요구**: 주장과 근거를 구분하도록 유도

### 어떤 학생에게 추천?
- 피상적인 답변에 그치는 학생
- 논리적 사고력을 키우고 싶은 학생
- 주장만 하고 근거를 제시하지 못하는 학생

### 활용 시나리오

#### 시나리오 1: 논리적 사고 훈련
**상황**: 학생이 "AI가 일자리를 없앨 것"이라고 주장

```
학생: "AI가 발전하면 일자리가 없어질 거야"
봇: "왜 그렇게 생각하게 됐어요?"

학생: "AI가 사람 일을 대체하니까"
봇: "모든 일을 대체할 수 있나요? 그 근거는?"

학생: "음... 단순 반복 작업은 대체되겠지만..."
봇: "그럼 '일자리가 없어진다'는 표현이 정확한가요?"

학생: "아, '일부 일자리는 변화한다'가 맞네요"
봇: "좋아요! 🎯 주장을 더 정확하게 다듬었네요."
```

#### 시나리오 2: 논리 오류 발견
**상황**: 학생이 성급한 일반화를 함

```
학생: "우리 반 친구 3명이 게임을 좋아하니까, 모든 학생이 게임을 좋아하는 거죠"
봇: "3명의 의견으로 전체를 판단할 수 있나요? 그 근거는?"

학생: "음... 표본이 너무 작은가요?"
봇: "좋은 지적이에요. 그럼 몇 명의 의견을 들어야 '모든 학생'이라고 말할 수 있을까요?"
```

### 교사 가이드

#### 사전 준비
1. **논쟁적 주제 선정**: 찬반이 나뉘는 주제 제시 (예: 스마트폰 사용, 숙제 필요성)
2. **인내심 필요**: 학생이 답을 찾기까지 시간이 걸릴 수 있음
3. **학습 목표**: "답 찾기"가 아니라 "사고 과정" 중요함을 강조

#### 사후 활동
1. **사고 과정 돌아보기**: 초기 주장 → 최종 결론 비교
2. **논리 구조 시각화**: 주장-근거-결론을 다이어그램으로 정리
3. **동료 토론**: 다른 학생과 같은 주제로 토론

#### 주의사항
- ⚠️ 학생이 "모르겠어요"라고 하면 방향 힌트 제공
- ⚠️ 지나치게 끈질기면 좌절할 수 있으니, 적절히 개입 필요
- ⚠️ 정답 압박이 아니라 사고 과정을 즐기도록 유도

## 3. 소통력 에이전트

### 기본 정보
- **테마 색상**: 분홍색 (Pink)
- **주요 역량**: 소통력 (Communication Clarity)
- **보조 역량**: 질문력 (Question Quality)
- **ID**: t1766120072135

### 핵심 특징
학생의 말이 **청자에게 어떻게 들릴지** 피드백하는 보조자입니다.
- **청자 중심 판단**: "상대방 입장에서 들으면..."
- **수정 제안 금지**: "이렇게 말해라"고 하지 않음
- **3단계 판정**: 이해 가능 / 오해 가능 / 이해 어려움

### 어떤 학생에게 추천?
- 의도와 다르게 전달되어 오해받는 학생
- 감정만 표현하고 구체적 요청을 못하는 학생
- 대인관계에서 갈등이 잦은 학생

### 활용 시나리오

#### 시나리오 1: 도움 요청 연습
**상황**: 학생이 선생님께 기한 연장을 요청하려 함

```
학생: "선생님, 저 좀 도와주세요."
봇: "이 말을 누구에게, 어떤 맥락에서 한 건가요?"

학생: "담임 선생님한테 수행평가 기한을 연장해달라고 말하는 거예요."
봇: "담임 선생님 입장에서 들으면, **이해 가능**합니다.
     도움을 요청하는 의도가 명확하고 예의 있게 들립니다.
     다만 구체적으로 무엇을 도와달라는 건지는 추가 설명이 필요할 수 있습니다."
```

#### 시나리오 2: 감정 표현 vs 요청 구분
**상황**: 학생이 부모님께 불만을 표현

```
학생: "부모님한테 '저 지금 너무 힘들어요 왜 이해를 못하세요 제 말 좀 들어주세요 진짜'라고 했어요."
봇: "부모님 입장에서 들으면, **이해 어려움**입니다.
     힘들다는 감정은 전달되지만, 구체적으로 무엇이 힘든지,
     무엇을 이해해달라는 건지, 어떻게 들어달라는 건지가 드러나지 않습니다.
     감정을 느끼는 것 자체는 문제가 아닙니다."
```

#### 시나리오 3: 친구 관계에서 오해
**상황**: 걱정하는 마음을 비난처럼 표현

```
학생: "친구한테 '너 진짜 답답하다'라고 말했어요. 친구가 시험 공부를 안 하고 놀기만 해서 걱정돼서요."
봇: "친구 입장에서 들으면, **오해 가능**합니다.
     걱정하는 의도가 있다면, 현재 표현은 비난이나 짜증으로 들릴 가능성이 있습니다.
     '답답하다'는 말이 감정 표현에 가깝고, 왜 걱정하는지는 드러나지 않기 때문입니다."
```

### 교사 가이드

#### 사전 준비
1. **역할극 준비**: 실제 상황을 시뮬레이션할 시나리오 준비
2. **안전한 환경**: 학생의 표현을 평가하지 않는 분위기 조성
3. **대상+맥락**: 반드시 "누구에게, 어떤 상황에서" 말하는지 명확히

#### 사후 활동
1. **재표현 연습**: 봇의 피드백을 듣고 다시 말해보기
2. **실제 적용**: 현실에서 개선된 표현을 사용해보고 결과 공유
3. **동료 피드백**: 같은 반 친구들에게도 들려주고 반응 확인

#### 주의사항
- ⚠️ 봇은 정답을 제시하지 않습니다 (학생이 스스로 수정)
- ⚠️ "사람마다 다르게 받아들일 수 있다"는 전제 유지
- ⚠️ 감정 표현 자체를 문제 삼지 않음

## 4. 질문력 에이전트

### 기본 정보
- **테마 색상**: 파란색 (Blue)
- **주요 역량**: 질문력 (Question Quality)
- **보조 역량**: 사고력 (Thinking Depth)
- **ID**: t1764659359295

### 핵심 특징
학생의 질문이 **5W1H가 명확해질 때까지** 문자 그대로만 답변합니다.
- **모호함 거부**: 불명확한 질문에는 최소한의 답변만
- **추측 금지**: 친절하게 보완해주지 않음
- **명확성 보상**: 질문이 명확해지면 완전한 답변 + 칭찬

### 어떤 학생에게 추천?
- 질문이 두루뭉술하여 원하는 답을 못 얻는 학생
- "이것" "그것" 같은 지시어를 남발하는 학생
- 구체적으로 표현하는 습관을 들이고 싶은 학생

### 활용 시나리오

#### 시나리오 1: 5W1H 완성하기
**상황**: 학생이 날씨를 물어봄

```
학생: "날씨 어때?"
봇: "어디?"

학생: "서울"
봇: "서울의 언제?"

학생: "지금"
봇: "오후 2시 서울은 맑고 15도입니다. 👍 장소와 시간을 모두 물어봤네요!"
```

#### 시나리오 2: 대명사 구체화
**상황**: 학생이 불명확한 지시어 사용

```
학생: "그거 어떻게 해요?"
봇: "무엇을?"

학생: "수학 문제요"
봇: "몇 쪽 몇 번?"

학생: "47쪽 3번 문제요"
봇: "47쪽 3번 문제는 이차방정식입니다. 인수분해 방법을 사용하면 됩니다.
     ✅ 페이지와 번호를 정확히 물어봤네요!"
```

#### 시나리오 3: 레벨별 힌트
**초급 (Lv.1)**: 많은 힌트 제공
```
학생: "이거 언제예요?"
봇: "무엇을 물어보는 건가요? 장소와 시간을 구체적으로 말해볼까요?"
```

**고급 (Lv.5)**: 최소 반응
```
학생: "이거 언제예요?"
봇: "다시 물어보세요"
```

### 교사 가이드

#### 사전 준비
1. **레벨 설정**: 학생 수준에 맞춰 힌트량 조절
2. **인내심**: 학생이 좌절하지 않도록 적절히 개입
3. **예시 제공**: 3회 이상 실패 시 구체적인 질문 예시 보여주기

#### 사후 활동
1. **Before/After 비교**: 초기 질문 vs 개선된 질문 비교
2. **동료 질문 평가**: 다른 학생의 질문이 명확한지 평가해보기
3. **현실 적용**: 실제 수업에서 선생님께 질문할 때 연습

#### 주의사항
- ⚠️ 지나치게 엄격하면 좌절할 수 있음
- ⚠️ 학생이 3회 이상 막히면 교사가 개입 또는 예시 제공
- ⚠️ 질문력 향상이 목적이지 정답 찾기가 아님을 강조

## 5. 실행력 에이전트

### 기본 정보
- **테마 색상**: 초록색 (Green)
- **주요 역량**: 실행력 (Execution Oriented)
- **보조 역량**: 소통력 (Communication Clarity)
- **ID**: t1766118428887

### 핵심 특징
학생의 실행 계획을 **SMART 기준**으로 점검하고 실행 가능한 수준으로 만듭니다.
- **한 번에 하나씩**: 한 가지 요소만 보완
- **SMART 내부 기준**: Specific, Measurable, Achievable, Relevant, Time-bound
- **실행 문장 완성**: "나는 [언제] [무엇을] 할 것이다"

### 어떤 학생에게 추천?
- 계획은 세우지만 실행으로 이어지지 않는 학생
- 추상적인 목표만 세우는 학생
- "열심히 하겠다", "노력하겠다" 같은 애매한 표현을 쓰는 학생

### 활용 시나리오

#### 시나리오 1: 추상적 계획 구체화
**상황**: 학생이 막연한 캠페인 계획을 제시

```
학생: "급식 남기지 않기 캠페인을 할 거예요."
봇: "그걸 실제로 하고 있는 장면을 떠올리면, 뭘 하고 있을까?"

학생: "포스터를 만들고 급식실에 붙일 거예요."
봇: "좋아. 그럼 언제 '했다'고 말할 수 있을까?"

학생: "포스터를 3장 만들어서 급식실 벽에 붙이면요."
봇: "그럼 이걸 언제 실행할지 하나만 고른다면?"

학생: "다음 주 화요일 점심시간 전까지요."
봇: "좋아. 그럼 지금까지 나온 내용으로 실행 문장을 네 말로 한번 만들어볼래?"

학생: "나는 다음 주 화요일 점심시간 전까지 급식 남기지 않기 포스터 3장을 만들어 급식실 벽에 붙일 것이다.
      실행 여부의 판단 기준은 포스터 3장이 급식실 벽에 붙어있는 것이다."
봇: "완성됐어. 이제 실행하면 돼."
```

#### 시나리오 2: 계획 크기 조절
**상황**: 학생이 너무 큰 계획을 세움

```
학생: "이번 주에 수학 문제집 100문제 풀 거예요."
봇: "이걸 다 하지 않는다면, 어디까지는 가능할까?"

학생: "음... 30문제요?"
봇: "그럼 그 30문제를 언제까지 풀 건지 정해볼까?"

학생: "매일 10문제씩 3일 동안이요."
봇: "좋아. 그럼 실행 문장을 만들어볼까?"
```

#### 시나리오 3: 측정 기준 만들기
**상황**: 학생이 확인 방법이 없는 계획을 제시

```
학생: "영어 공부를 열심히 할 거예요."
봇: "언제 '했다'고 말할 수 있을까?"

학생: "음... 단어를 외우는 거요?"
봇: "끝났다는 걸 보여줄 수 있는 흔적이 있을까?"

학생: "단어장에 체크 표시하는 거요!"
봇: "좋아. 몇 개를 체크하면 '했다'고 할 수 있을까?"
```

### 교사 가이드

#### 사전 준비
1. **실행 가능성 중시**: "멋진 계획"보다 "실행 가능한 계획" 강조
2. **작은 성공 경험**: 처음에는 쉬운 목표부터 시작
3. **SMART 이해**: 교사가 먼저 SMART 기준 숙지

#### 사후 활동
1. **실행 추적**: 학생이 실제로 실행했는지 확인
2. **성공/실패 분석**: 왜 실행했거나 못했는지 돌아보기
3. **다음 계획**: 성공 경험을 바탕으로 다음 실행 계획 세우기

#### 주의사항
- ⚠️ 봇은 계획의 성공/실패를 평가하지 않음 (교사 영역)
- ⚠️ "더 나은 방법"을 제안하지 않음
- ⚠️ 학생이 스스로 실행 문장을 완성하도록 유도

## 공통 활용 팁

### 1. 세션 시간 관리
- **권장 시간**: 15-20분 (최대 30분)
- **이유**: 집중력 유지 및 피로 방지

### 2. 교사 역할
- **관찰자**: 학생-봇 대화를 지켜보며 학습 패턴 파악
- **개입 시점**: 학생이 3회 이상 막히거나 좌절할 때
- **최종 평가**: 봇이 평가하지 않는 부분을 교사가 피드백

### 3. 학생 준비사항
- **명확한 주제**: "뭘 할지 모르겠어요"로 시작하지 않기
- **충분한 시간**: 급하게 하지 말고 여유 있게 접근
- **개방적 태도**: 봇의 질문을 귀찮아하지 않기

### 4. 효과 측정
- **역량 점수**: 30일 후 해당 역량 점수 변화 확인
- **행동 변화**: 실제 수업/생활에서 변화 관찰
- **자기 보고**: 학생 스스로 느끼는 변화 기록

## FAQ

### Q1: 여러 봇을 동시에 사용해도 되나요?
**A**: 가능하지만, 한 번에 하나씩 집중하는 것을 권장합니다. 역량이 골고루 부족하다면, 가장 낮은 역량부터 시작하세요.

### Q2: 봇이 답을 주지 않아서 답답해요
**A**: 이 봇들은 "정답 제공"이 아니라 "사고 훈련"이 목적입니다. 교사가 사전에 학생에게 목적을 명확히 설명해주세요.

### Q3: 학생이 봇과의 대화를 거부하면?
**A**: 강제하지 말고, 먼저 교사가 시연해 보이거나, 친구와 함께 사용하게 하세요. 게이미피케이션 요소(뱃지, 레벨업)를 활용하는 것도 방법입니다.

### Q4: 봇 추천 기능은 어떻게 작동하나요?
**A**: 학생의 역량 점수를 분석하여 가장 낮은 역량을 개선할 수 있는 봇을 자동 추천합니다. Dashboard에서 확인할 수 있습니다.

### Q5: 역량 점수는 어떻게 계산되나요?
**A**: 학생-봇 대화 내용을 AI가 자동 분석하여 6가지 역량을 평가합니다. 30일 가중 평균으로 계산되며, 초기 진단 결과도 반영됩니다.
